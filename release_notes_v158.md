## v1.5.8 — 深度暗色模式适配 & 极致消除白闪

本版本重点解决了暗色模式下应用启动、页面切换以及网站加载初期的所有“白闪”问题，提供了更纯净的夜间阅读体验。

### ✨ 主要改进

- **🚀 零延迟启动主题**：
  - 重构了启动逻辑，App 现在会在 `installSplashScreen` 之前预读配置并应用 `Theme.App.Starting.Dark`。
  - 确保从点击图标到进入主界面的整个过程中，窗口背景始终保持纯黑。

- **🌙 原生级强制暗黑 (Force Dark)**：
  - 引入 `androidx.webkit` 支持，开启 WebView 原生 `FORCE_DARK`。
  - 采用 `USER_AGENT_DARKENING_ONLY` 激进渲染策略，使浏览器内核在解析 HTML 初期即自动完成深色转换。

- **⚡ 消除网站加载页白屏 (Loading Page)**：
  - 在 `WebChromeClient` 中实现了高频探测逻辑。
  - 当页面加载达到 **2%** 时，即提前注入暗色滤镜 CSS，完美解决了网站自带 Loading 界面（带 Logo 的白色全屏页）无法被后期脚本覆盖的顽疾。

- **🎨 UI 细节打磨**：
  - **状态栏适配**：使用 `WindowInsetsControllerCompat` 动态切换状态栏图标颜色，确保在黑色背景下时间、电量等图标清晰可见。
  - **背景同步**：统一了所有 WebView（包括后台加载页）和根布局的初始背景色设置。

- **🛡️ 脚本健壮性**：
  - 优化了暗色 CSS 注入脚本，支持在 `document.head` 生成前自动挂载至 `documentElement`，消灭任何渲染死角。

### 🏎️ 核心性能优化 (漫画加载)

- **漫画 URL 提取策略重构**：
  - 彻底废弃了依赖网页滚动条位置 `.comicIndex` 的旧版探测逻辑（该逻辑在后台隐藏 WebView 中因渲染滞后而极其缓慢且不可靠）。
  - **全新计数器模式**：直接通过监控 DOM 中已加载 `data-src` 的图片数量来精准上报进度。
  - **极致加载速度**：解决了长章节（几百页）加载到一半就卡住或进度虚标的问题，现在后台提取速度可完美适配极速滑动，确保所有页面 URL 瞬间捕获。

- **🏎️ 后台 URL 提取动态限速策略 (Adaptive Extraction)**：
  - **智能调速引擎**：在 `h.js` 中实现了自适应限速算法。App 会根据网页图片加载的实时吞吐量，动态调节后台模拟滚动的跨度与频率。
  - **快慢自适应**：当网络波动导致加载缓慢时，脚本会自动降速“等待”，避免因过度滚动导致漏页；当网络环境优越时，则会自动提速（最高 600px/帧）进入冲刺模式。
  - **帧级优化**：采用 `requestAnimationFrame` 进行驱动，在保证提取效率的同时，极大降低了后台 WebView 对系统资源的占用，确保前台阅读无微卡顿。
